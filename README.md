# 项目计划


### 总体调整思路
- **扩散模型**：不再在原始像素/文本上做扩散，改为在 **低维嵌入空间** 进行扩散生成（类似 DiffRec 的轻量版本），参数量可控制在百万级。
- **多模态对齐**：使用预训练的轻量 CLIP（如 ViT-B/32）一次性提取特征，后续只训练线性投影层。
- **RL 微调**：采用 **离线 DPO**，避免在线交互和环境模拟，大幅降低训练开销。
- **知识图谱**：利用结构化属性 + 小型图数据库（如 NetworkX）完成路径推理，无需训练 GNN。

---

## 第一阶段：多模态数据处理（简化）

**目标**：提取轻量、低维的多模态特征，构建用户行为序列。

1.  **数据集选择与裁剪**
    - 使用 Amazon Review 2023 子集（例如只保留 **Clothing, Shoes, Jewelry** 大类），过滤交互次数≥3的用户和商品，将数据集大小控制在 **10万商品、5万用户** 以内。数据链接：https://amazon-reviews-2023.github.io/
    - 每件商品保留 **主图**（若多角度图可用，随机选一张即可）和 **标题+描述前100词**。

2.  **特征提取（离线，复用预训练模型）**
    - **图像特征**：使用 **CLIP ViT-B/32**（约 150M 参数）提取主图特征，输出 512 维向量。
    - **文本特征**：使用 **DistilBERT**（或 CLIP 文本编码器）提取标题+描述的嵌入，同样输出 512 维。
    - **ID 嵌入**：随机初始化的商品 ID 嵌入（维度 128）。
    - **存储**：将三类特征存储为 `.npy` 文件，按商品 ID 索引。

3.  **用户行为序列**
    - 按时间排序每个用户的购买/点击序列，截断长度 L=20（不足则填充）。
    - 构建 `(user_id, sequence_of_item_ids)` 数据集，用于后续序列建模。

## 第二阶段：轻量级扩散模型预训练（CCDRec 简化版）

**核心思路**：用 **DiffRec**（Diffusion for Recommendation）的思想，在协同嵌入空间做扩散，不再引入复杂的多模态对齐模块，而是直接用对比学习对齐多模态与协同信号。

### 2.1 DMA 多模态对齐（轻量版）
- 不使用交叉注意力，而是对每个商品，将其图像特征 \(e_i\)、文本特征 \(e_t\)、ID 嵌入 \(e_c\) 拼接，经过一个 **两层的 MLP**（输入 512+512+128=1152，输出 256）得到统一表示 \(h\)。
- **对比损失**：同批次内，同一商品的不同模态应靠近，不同商品的表示应推开。使用 InfoNCE 损失，batch size 设为 256 即可。

### 2.2 NDI 负样本渐进合成（简化）
- 不通过扩散生成负样本，而是采用 **基于难度的负采样**：
    1. **简单负样本**：随机从所有商品中采样。
    2. **中等难度**：选择与正样本在图像/文本特征上相似但 ID 不同的商品（用 Faiss 快速检索）。
    3. **困难负样本**：选择与正样本有相似属性（从标题中提取的关键词）但交互历史不同的商品。
- 训练时，按 **课程学习** 顺序：先只用简单负样本，逐步混入中等、困难负样本。

### 2.3 扩散模型（轻量版）
- **模型结构**：使用 **MLP 作为去噪网络**（3 层，隐藏层 512），输入为加噪的表示 \(h_t\) 和时间步 \(t\)，输出预测的噪声。
- **扩散过程**：仅对商品的统一表示 \(h\) 进行前向加噪和反向去噪。这意味着我们训练一个 **生成模型**，能够从噪声中重建出商品的表示。
- **损失函数**：
    - **ELBO 损失**（MSE between predicted noise and actual noise）
    - **对比损失**（用于对齐多模态，与上述 InfoNCE 共享）
    - 总损失 = ELBO + λ * InfoNCE

**训练细节**：
- 模型参数量约 20M，单卡可轻松运行。
- 使用 AdamW，学习率 1e-4，batch size 128，训练 50 个 epoch。

## 第三阶段：强化学习微调（Curriculum DPO 简化）

**目标**：在扩散预训练模型基础上，用离线偏好优化进一步提升推荐质量。

### 3.1 奖励函数简化
- 由于无法训练复杂的 CTR 模型，我们使用 **可离线计算的代理指标**：
    - **\(R_{ctr}\)**：用用户历史行为序列中商品的流行度（点击次数）作为替代，鼓励推荐热门商品。
    - **\(R_{compat}\)**：计算推荐商品与用户历史商品在统一表示空间中的余弦相似度（用训练好的 MLP 输出的 256 维表示）。
    - **\(R_{div}\)**：推荐列表内商品表示的方差（多样性）。
- 最终 \(R = R_{ctr} + 0.5 \times R_{compat} + 0.3 \times R_{div}\)。

### 3.2 离线 DPO（Direct Preference Optimization）
- **生成偏好对**：对每个用户，用当前模型（预训练扩散模型）采样 N 个推荐列表，计算每个列表的奖励，取最高和最低的作为 chosen 和 rejected。
- **训练**：固定扩散模型中的去噪网络，仅微调一个 **策略头**（一个线性层），优化 DPO 损失。
- **课程策略**：不采用复杂的课程，而是先用 **简单样本**（用户历史短，商品少）训练 10 轮，再用全量数据训练。

**资源消耗**：DPO 训练仅需 1-2 小时，内存占用小。

## 第四阶段：可解释性增强（轻量版）

### 4.1 构建属性知识图谱
- **实体**：商品（ID）、品牌（从标题中抽取）、品类（如“连衣裙”、“运动鞋”）、颜色（用正则提取）、风格（基于 CLIP 聚类，将图像特征聚为 20 类作为“风格”）。
- **关系**：
    - `商品 - 属于品牌 - 品牌`
    - `商品 - 具有属性 - 品类/颜色/风格`
    - `商品 - 相似 - 商品`（基于图像特征余弦相似度 > 0.9 构建）
- **存储**：使用 **NetworkX**（内存图）或 SQLite，无需图数据库。

### 4.2 推理路径生成
- 当模型推荐商品 B 时，在图中寻找从用户最近购买的 3 个商品到 B 的最短路径（限制长度 ≤3）。
- **路径示例**：`用户买过 A（运动鞋）` → `A 属于品牌 Nike` → `Nike 有商品 B（运动裤）` → 推荐 B。
- **自然语言生成**：用模板将路径转成解释，例如：“因为你购买了 [A]（同品牌 [Brand]），所以我们推荐这款 [B]。”

## 与原方案对比：保留了哪些核心思路？

- **多模态对齐**：仍使用对比学习将图像、文本、ID 拉近，只是投影层更浅。
- **扩散生成**：在嵌入空间进行扩散，保留了“生成高质量候选”的能力。
- **负样本渐进**：用采样难度替代生成难度，实现了课程学习思想。
- **RL 微调**：用 DPO 实现偏好优化，无需在线交互。
- **可解释性**：构建知识图谱并做路径推理，只是规模更小。
